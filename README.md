# TimeScreen Control — Родительский контроль для Windows

Программа для ограничения времени использования компьютера. Двухкомпонентная архитектура: служба Windows (SYSTEM, неубиваемая) + пользовательский GUI-агент. Не требует Python на целевом ПК.

## Возможности

- ✅ **Графический интерфейс** — вкладки «Настройки», «Управление», «Система», без консоли
- ✅ **Служба Windows** — мониторинг времени под SYSTEM, нельзя убить через Task Manager
- ✅ **Гибкое расписание** — несколько интервалов `ЧЧ:ММ-ЧЧ:ММ` (включая через полночь)
- ✅ **Блокировка всех мониторов** — `EnumDisplayMonitors`, каждый экран получает полноэкранное окно
- ✅ **Таймер-индикатор** — в углу экрана, без мерцания (полупрозрачный, без `transparentcolor`)
- ✅ **Авторазблокировка** — экран сам закроется, когда настанет разрешённое время
- ✅ **Grace-период** — 10 минут без повторной блокировки после ввода пароля
- ✅ **Разблокировка по паролю** — SHA-256 хэширование с контролем целостности конфига
- ✅ **Admin Unlock** — вход как администратор Windows сбрасывает блокировку
- ✅ **Multi-user** — настраиваемый список контролируемых пользователей Windows
- ✅ **Защита от взлома** — проверка хэша конфига, при повреждении — перманентная блокировка
- ✅ **Watchdog** — служба перезапускает убитого агента; агент перезапускает упавшую службу
- ✅ **Без зависимостей** — на целевом ПК не нужен Python, права администратора, интернет

## Архитектура (два EXE)

```
TimeScreenControl.exe              ← GUI, оконный (--windowed, --onefile)
  ├── (default)                    → главное окно настроек (три вкладки)
  ├── --user-agent                 → пользовательский агент (индикатор + мониторинг флага)
  ├── --lock                       → полноэкранная блокировка
  ├── --timer                      → таймер-индикатор в углу экрана
  ├── install-user                 → установка для текущего пользователя
  ├── uninstall-user               → удаление
  └── --install-service            → установка службы
  └── --recovery                   → сброс конфига при повреждении

TimeScreenService.exe              ← Служба, консольная (--onedir)
  └── Запускается SCM как SYSTEM в Session 0
      Регистрируется через StartServiceCtrlDispatcherW (ctypes)
      Пишет lock_flag в %PROGRAMDATA%\TimeScreen\
      Перезапускает агента при убийстве (через schtasks)
```

### Почему --onedir для службы

PyInstaller `--onefile` bootloader перезапускает сам себя (меняет PID) для ASLR на Windows 10/11. SCM отвергает подключение с другого PID → Error 1053. `--onedir` не перезапускается, PID сохраняется.

## Быстрый старт

### На компьютере разработчика (где есть Python):

```bash
cd ParentalControlStandalone

# 1. Собрать GUI .exe (оконный)
python -m PyInstaller --onefile --windowed --name "TimeScreenControl" ^
    --hidden-import=tkinter --hidden-import=tkinter.messagebox ^
    --hidden-import=tkinter.ttk --hidden-import=tkinter.simpledialog ^
    parental_control_gui.py

# 2. Собрать Service .exe (консольный --onedir)
python -m PyInstaller --onedir --name "TimeScreenService" ^
    service_daemon.py

# 3. Скопировать в установочный пакет
copy dist\TimeScreenControl.exe TimeScreen_Installer\
xcopy /E /I dist\TimeScreenService TimeScreen_Installer\TimeScreenService
```

Готовый пакет: `TimeScreen_Installer\` — скопируйте эту папку на целевой компьютер.

### На целевом компьютере (Python не нужен):

1. Скопируйте папку `TimeScreen_Installer` на целевую машину
2. Запустите `install.bat` (НЕ от админа — установка пользователя)
3. На рабочем столе появятся ярлыки:
   - **TimeScreen — Настройки** (главное окно)
   - **TimeScreen — Защита** (вкл/выкл)
4. Монитор времени добавится в автозагрузку

### Настройка:

1. Откройте «TimeScreen — Настройки»
2. Вкладка **Настройки**: задайте пароль, расписание, список пользователей
3. Вкладка **Управление**: нажмите **«Запустить защиту»**
4. Вкладка **Система**: нажмите **«Установить службу»** (запросит UAC)
5. Проверьте: `sc query TimeScreenControl` → `STATE: 4 RUNNING`

### Полная очистка:

```batch
force_cleanup.bat   ← удаляет службу, процессы, файлы, ярлыки (запустить от админа)
```

## Как это работает

```
┌─────────────────────────────────────────────────┐
│  СЛУЖБА (TimeScreenService.exe, SYSTEM)         │
│  Session 0, нельзя убить через Task Manager      │
│                                                   │
│  Каждые 5 сек:                                   │
│  1. Читает pc_config.json                        │
│  2. Проверяет время                               │
│  3. Если запрещено → пишет lock_flag             │
│  4. Если разрешено → удаляет lock_flag           │
│  5. Каждые 30 сек: проверяет жив ли агент         │
│     Если убит → перезапускает через schtasks      │
└──────────────┬──────────────────────────────────┘
               │ lock_flag
               ▼
┌─────────────────────────────────────────────────┐
│  АГЕНТ (TimeScreenControl.exe --user-agent)      │
│  Сессия пользователя, GUI                        │
│                                                   │
│  Каждые 3 сек:                                   │
│  1. Проверяет lock_flag                          │
│  2. Если есть → запускает LockScreen на всех     │
│     мониторах и ждёт закрытия                    │
│  3. Если нет → продолжает показывать индикатор   │
│                                                   │
│  Индикатор (--timer):                             │
│  - Зелёный: обратный отсчёт до блокировки         │
│  - Жёлтый: grace-период                           │
│  - Красный: компьютер заблокирован                │
│                                                   │
│  LockScreen (--lock):                             │
│  - Каждые 3 сек проверяет авторазблокировку      │
│  - Если время стало разрешённым → самозакрытие   │
└─────────────────────────────────────────────────┘
```

## Файлы и расположение

После установки:

```
%LOCALAPPDATA%\TimeScreen\
├── TimeScreenControl.exe       — GUI (оконный)
├── run_agent.vbs               — VBS для скрытого запуска агента
└── TimeScreenService\          — Служба (--onedir)
    ├── TimeScreenService.exe
    └── _internal\               — Python runtime + зависимости

%PROGRAMDATA%\TimeScreen\
├── pc_config.json              — конфиг (пароль, расписание, пользователи)
├── lock_flag                   — флаг блокировки (1 = заблокирован)
├── monitor.pid                 — PID службы
├── agent.pid                   — PID агента
├── service.log                 — журнал работы
└── run_agent.vbs               — VBS для службы (перезапуск агента)
```

Ярлыки:
- `%USERPROFILE%\Desktop\TimeScreen - Настройки.lnk`
- `%USERPROFILE%\Desktop\TimeScreen - Защита.lnk`
- `%APPDATA%\...\Startup\TimeScreen.lnk` (автозагрузка)

## Решение проблем

### Служба не запускается (Error 1053)
- Убедитесь, что `TimeScreenService.exe` собран с `--onedir` (не `--onefile`)
- Проверьте что `_internal\` лежит рядом с `TimeScreenService.exe`
- Запустите `force_cleanup.bat` и переустановите службу

### Служба висит в START_PENDING
- Старая служба удалена не полностью → перезагрузите ПК
- Или удалите через реестр: `HKLM\SYSTEM\CurrentControlSet\Services\TimeScreenControl`

### Индикатор не показывается
- Агент не запущен → нажмите «Запустить защиту» в GUI
- Или запустите вручную: `TimeScreenControl.exe --user-agent`

### Мерцание экрана
- Обновитесь до последней версии (исправлено: `-alpha` вместо `-transparentcolor`, `CREATE_NO_WINDOW` для subprocess)

### Блокируется только один монитор
- Обновитесь — используется `EnumDisplayMonitors`

### Забыли пароль
- Удалите `%PROGRAMDATA%\TimeScreen\pc_config.json` и задайте пароль заново
- Или Admin Unlock (кнопка на экране блокировки)
